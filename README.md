# Paper Desk

A web app for students to **practice trading with entirely simulated money and simulated
market data** — spot equities, options (with Greeks), futures (with margin and daily
settlement), FX (with Sales and Trader desk views), forwards, and a teaching
interest-rate swap.

> **Nothing here is real.** No brokerage, no market data feed, no real money.
> Prices are generated by a seeded geometric-Brownian-motion simulator; this is a
> teaching tool, not a broker.

## Stack

| Layer | Tech |
|---|---|
| Frontend | TypeScript, React 18, **webpack 5** (direct config, no CRA/Vite), Tailwind CSS |
| Frontend state | **esp-js** ecosystem instead of Redux: `esp-js` (Router), `esp-js-polimer` (state models), `esp-js-react` (view bindings), `esp-js-di` (DI container) |
| Realtime | STOMP over WebSocket (`@stomp/stompjs` ⇄ Spring WebSocket) |
| Charts | `lightweight-charts` |
| Backend | Java 21, Spring Boot 3 (Web, Security + JWT, WebSocket, Data JPA), Maven |
| Database | Oracle-compatible schema via Flyway; **H2 in Oracle mode** for dev/tests; real Oracle is config-only (see below) |

## Quick start

```bash
# terminal 1 — backend (http://localhost:8080)
cd backend && mvn spring-boot:run

# terminal 2 — frontend dev server (http://localhost:3000, proxies /api and /ws)
cd frontend && npm install && npm run dev
```

Open http://localhost:3000, sign up (student or instructor), pick a scenario
("Calm market", "Bull run", "High volatility", "Crash") and trade. Use the header
clock controls — **pause**, **⏭ +1 day** (runs daily settlement, expiries, fixings
instantly) and the acceleration selector — to teach theta decay and margin mechanics
without waiting.

### Tests

```bash
cd backend && mvn test        # pricing (Black-Scholes/GK vs textbook values, put-call
                              # parity), GBM determinism, fills, margin, expiry, fixings
cd frontend && npm test       # polimer model reducers + esp devtools tracer
cd frontend && npm run typecheck && npm run build
```

## How the simulation works

- **Sim clock**: server-authoritative, per session, accelerated (default 300× —
  one sim day ≈ 4.8 real minutes), pausable and steppable by day.
- **Prices**: geometric Brownian motion per symbol on a fixed 5-sim-minute grid,
  RNG seeded from `(scenario seed, symbol)` — the same scenario always replays the
  identical market, so instructors can grade/compare students on equal footing.
- **Scenarios** modulate drift/vol per symbol; "Crash" schedules a −25% jump with a
  volatility spike at sim day 10.
- **Derivative pricing**: Black-Scholes(-Merton) with carry for equity options,
  Garman-Kohlhagen for FX options (Greeks: delta, gamma, theta/day, vega/1%),
  cost-of-carry for futures/forwards, and a deliberately simplified flat-curve swap
  PV against a seeded mean-reverting floating index.
- **Matching**: no order book — market orders fill at mid ± a small spread, limit
  orders rest and fill when the simulated price crosses.
- **Daily roll** (each sim midnight): futures are marked to market with variation
  margin posted to cash (margin call → forced liquidation if cash goes negative);
  expiring options auto-exercise ITM (cash-settled intrinsic) or expire worthless;
  forwards settle at maturity; swap fixings pay monthly; option chains roll new
  expiries in; per-account equity snapshots feed the day-P&L and leaderboards.

## Product surfaces

- **Dashboard** — portfolio value, cash, margin held, today's P&L, equity curve,
  desk event feed (fills / margin calls / settlements pushed over WebSocket).
- **Market** — grouped watchlist (equities, FX, futures, forwards, swaps) with live
  simulated prices, candle chart per symbol, order ticket (market/limit; margin
  requirement shown for futures before confirming).
- **Options Chain** — strikes × expiries with live premiums and full Greeks;
  click a premium to load the ticket.
- **FX Sales** — client RFQ for custom-strike/expiry FX options: dealer mid + sales
  margin = all-in client price; executing books the deal (`view_context=SALES`).
- **FX Trader** — dealer mids, the FX book with aggregated Greeks, and a spot risk
  ladder (P&L/Δ/Γ/vega across ±3% spot shifts).
- **Portfolio** — live marks, unrealized/realized P&L, per-position Greeks, margin
  usage, settlement/lifecycle history.
- **Blotter** — every order with fills, statuses, reject reasons, sales/trader tags.
- **Classroom** — instructors create cohorts (dedicated seeded session + join code
  + starting balance); students join; leaderboard ranks by equity with return % and
  max drawdown.

## Gamification — XP, levels & badges

Learning is nudged with a lightweight progression system, consistent across the app:

- **XP**: every fill earns XP; badges award larger one-time chunks. The header shows
  your level and XP bar on every tab.
- **Levels**: Observer → Apprentice → Associate → Trader → Senior Trader → Desk Head
  → Rainmaker → Market Wizard.
- **Badges as curriculum**: the 15 badges walk students through the syllabus — first
  trade per instrument type, a filled limit order, diversification, a profitable day,
  a 5-day hot streak, and the risk lessons ("Margin Call Survivor", "Theta Tuition",
  "In The Money", "Crash Survivor").
- **Progress tab**: level roadmap plus earned/locked badge grid; unlocks toast
  instantly on any tab (pushed over the account WebSocket topic).
- **Classroom tie-in**: the leaderboard shows each student's level alongside equity,
  return % and drawdown.

## esp-js DevTools (dev mode)

A **reusable, decoupled esp-js event tracer** lives in `frontend/src/devtools/` —
it depends only on `esp-js`, not on any app code, so it can be dropped into any
esp-js application:

```ts
import {installEspDevTools} from './devtools';
const devTools = installEspDevTools(router, {ignoredEvents: ['someChattyTick']});
```

- Records every `publishEvent`/`broadcastEvent` (model id, event type, payload)
  plus the post-dispatch model state snapshot.
- **Redux DevTools bridge**: with the browser extension installed, each esp model
  appears as a store (`esp:tradingModel`, …) and events stream in as actions with
  state — inspect esp flow in the familiar UI.
- **In-app overlay**: toggle with **Ctrl+Shift+E** (or the floating `esp ⚡` badge) —
  live event log with filtering and a payload/state inspector.
- Installed only when `NODE_ENV === 'development'`; production builds carry none of it.

(esp-js-polimer also ships its own `enableReduxDevTools` per model; this plugin is
router-level and framework-agnostic by design, tracing *all* events across models.)

## Using real Oracle instead of H2

The schema (Flyway, `backend/src/main/resources/db/migration`) is written
Oracle-compatible and the Oracle JDBC driver ships with the build. Point the
datasource at Oracle and start:

```bash
docker compose up -d oracle   # gvenzl/oracle-free (23ai)
SPRING_DATASOURCE_URL=jdbc:oracle:thin:@//localhost:1521/FREEPDB1 \
SPRING_DATASOURCE_USERNAME=paperdesk \
SPRING_DATASOURCE_PASSWORD=paperdesk \
mvn spring-boot:run
```

## Teaching simplifications (by design)

- Continuous 24/7 market (no weekends/holidays), one flat rate curve, implied vol =
  the symbol's realized vol parameter.
- Options cash-settle intrinsic at expiry (no physical delivery/pin risk).
- Shorting spot/options requires no margin; futures margin is initial/maintenance
  with a simple liquidate-on-negative-cash rule.
- Forwards/swaps are uncollateralized OTC teaching instruments.
- WebSocket topics are unauthenticated (simulated data only); JWT secret in
  `application.yml` is a dev default.
